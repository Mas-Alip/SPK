Project Journal
===============

Purpose
-------
This journal records development activities, decisions, and important events for the project.

How to use
---------
- Keep a chronological `journal/` folder with one markdown file per date (YYYY-MM-DD.md).
- Use this `JOURNAL.md` as the canonical index and template.
- Each entry should include: Date, Author, Summary, Files changed, Tests/Validation, Issues, Next steps.

Template for entries
-------------------
Date: YYYY-MM-DD
Author: Your Name
Purpose: Short description why this entry

Summary:
- Bulleted summary of what changed or was discovered.

Files changed / added:
- path/to/file.py — short description

Testing / Validation:
- What you ran and the results (unit tests, manual steps, outputs)

Issues / Notes:
- Any problems found, decisions made, trade-offs, TODOs.

Next steps:
- Concrete small tasks to continue.

Index of entries
----------------
- 2025-09-29 — Initial AHP/SAW refactor & AHP Full implementation (see journal/2025-09-29.md)


Guidelines / Tips
-----------------
- Write brief, technical notes; include commands and small snippets where helpful.
- Link to ticket/issue numbers if you have a tracker.
- Keep each entry focused on a single day or topic.
